Update and detach lifecycle configurations

A lifecycle configuration script can't be changed after it's created. To update your script, you must create a new lifecycle configuration script and attach it to the respective domain, user profile, or shared space. For more information about creating and attaching the lifecycle configuration, see Create and associate a lifecycle configuration.

The following topic shows how to detach a lifecycle configuration using the AWS CLI and SageMaker console.

Prerequisites

Before detaching a lifecycle configuration, you must complete the following prerequisite.

  • To successfully detach a lifecycle configuration, no running application can be using the lifecycle configuration. You must first shut down the running applications as shown in Shut Down and Update SageMaker Studio and Studio Apps.

Detach using the AWS CLI

To detach a lifecycle configuration using the AWS CLI, remove the desired lifecycle configuration from the list of lifecycle configurations attached to the resource and pass the list as part of the respective command:

  • update-user-profile

  • update-domain

  • update-space

For example, the following command removes all lifecycle configurations for KernelGateways attached to the domain.

aws sagemaker update-domain --domain-id domain-id \
--region region \
--default-user-settings '{
"KernelGatewayAppSettings": {
  "LifecycleConfigArns":
    []
  }
}'

Detach using the SageMaker console

The procedure to detach a lifecycle configuration from the SageMaker console depends on the type of resource that you are detaching the lifecycle configuration from. You can't detach a lifecycle configuration from a shared space by using the SageMaker console. For that, you must use the AWS CLI.

Detach from a domain

  1. Open the Amazon SageMaker console at https://console.aws.amazon.com/sagemaker/.

  2. On the left navigation pane, choose Domains.

  3. From the list of Domains, select the name of the Domain to detach the lifecycle configuration from.

  4. On the Domain details page, choose the Environment tab.

  5. Under Lifecycle configurations for personal Studio apps, select the lifecycle configuration to detach.

  6. Choose Detach. This opens a pop-up window.

  7. From the pop-up window, choose Detach.

Detach from a user profile

  1. Open the Amazon SageMaker console at https://console.aws.amazon.com/sagemaker/.

  2. On the left navigation pane, choose Domains.

  3. From the list of Domains, select the name of the Domain that contains the user profile to detach the lifecycle configuration from.

  4. On the Domain details page, choose the User profiles tab.

  5. From the list of user profiles, select the name of the user profile to detach the lifecycle configuration from.

  6. From the User Details page, choose Edit.

  7. On the left navigation, choose Studio settings.

  8. Under Lifecycle configurations attached to user, select the lifecycle configuration to detach.

  9. Choose Detach. This opens a pop-up window.

  10. From the pop-up window, choose Detach.
